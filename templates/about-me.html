Programming Philosophy


Short But Frequent Structures

Short But Frequent Structures
Functions and classes should be very small. This increases the readability of code and reduces the learning curve required to understand it.1

No Horizontal Scrolling

No Horizontal Scrolling
Lines of code should never be so wide that horizontal scrolling or zooming out is required. Keeping lines short not only makes it easier to navigate the code, it also vastly increases readability.1

Self-Documenting Code with Good Naming

Self-Documenting Code with Good Naming
Functions, classes, and variables should have descriptive and meaningful names. Ideally, with good enough naming, no comments (other than legal information and TODOs) will be necessary.1
